HTML Typography, Lists, Links

The "p" tag is one of the most basic typography tags, and is used to separate paragraphs. This very text is contained inside a <p>. Notice that paragraph elements will by default space themselves out. The gaps that appear between them vertically are the result of top and bottom margins (margins are outer spacing, applied around an element). Now, margins are not strictly speaking HTML; they are actually CSS properties, which are applied on certain HTML elements. But, HTML and CSS are used together, and so most HTML elements have certain default CSS properties. Case in point, the h1 tag shown above comes with a default font size, and - just as the p tag - margins at the top and bottom.

If you're using the Google Chrome browser, you can actually check what CSS properties are applied on each element. Go ahead and right-click on the h1 above, and click "Inspect" (in the browser window, not the code). A Chrome DevTools (Developer Tools) panel should show up. Within this panel, under the "Styles" tab (should be on the right), you should see references to CSS properties such as display, font-size, font-weight, and margins. Those are the defaults that the h1 tag has.

When we write our own CSS code, we can choose to override these default properties, and apply our own. The properties listed under the "Styles" tab of the h1 tag are just a few of the hundreds of properties that CSS provides us with. With this large set of styling options, we make our HTML elements look any way we want.

Let's observe how I have used some of the other HTML tags at the top.

I have been using <hr> to seperate different sections of content. In the book, the "hr" tag also includes a slash (as does the "br" tag). This used to be required in the past, but is no longer necessary in newer versions of HTML. So, we can omit the slash ("/"). Note also that both, "hr" and "br" are self-closing elements (they're composed of an opening tag, but no closing tag).

In the above paragraph, I used "code" tags, to display the syntax of the "hr" tag (I also did the same thing for "p" tags in the first paragraph). These "code" tags will by default change the font of the text they wrap around to a font more reminiscent of code editors or the command line. They also carry semantic meaning: their content is computer code.

Here's another example, this time of a two-line code block (you don't need to understand the JavaScript code between the code tags):

let message = "Hello World";
console.log(message);

You'll observe that in cases where I displayed the "p" and "hr" tags within angle brackets, they are prefaced with "[ampersand]lt;" and suffixed with "[ampersand]gt;". These two strange sets of symbols are examples of what we call "HTML entities", and they are rendered by the browser as less-than and greater-than characters (or left and right angle brackets, if you prefer). The reason why I didn't just use the literal angle brackets in their place is that angle brackets are registered HTML syntax. Whenever you use an angle bracket within an HTML document, the browser assumes that you're writing a tag, and so it doesn't render the character on the page literally. So, if you want to display an angle bracket directly, you need a workaround - which HTML entities provide.

Most often, however, entities are used to display characters on the page for which we don't have keyboard keys, such as the registered trademark ("©"). You can see more examples of HTML entities here.

Let's go over bold and italic styling. There are two main ways to make text bold: using "b" tags or using "strong" tags. Functionally, both are the same, but "strong" tags indicate strong emphasis, while "b" tags are purely stylistic.

There are also two main ways of making text italic: using "i" tags or using "em" tags. They are also both functionally identical, and their difference comes down to semantics: "em" tags indicate emphasis, while "i" tags are purely stylistic.

Links can be added to a page using the anchor ("a") tag. We simply wrap the text that we want to turn into a link with these "a" tags, and add the "href" attribute in order to tell the browser where the link should point to. This link points to Google. By default it will appear blue and underlined, but once you click it, and go back to the page where the link was, it will have turned purple. This is because links have states. Once a link has been clicked, it is marked as "visited". Another important link state is "hover". It controls the appearance of the link when a mouse is hovering over it (this is how some websites change a link's color on hover). We'll see how to take full adventage of these link states once we start learning CSS.

If you want a link to open in a seperate page, you can add the "target" attribute and give it a value of "_blank", like this. There are other values that can be supplied to this target attribute ("_self") is the default, but they are rarely used in practice.
